# Paper Outline Doctor

把一篇论文该怎么写、先写什么、哪些章节要等结果，拆成可执行的写作计划。

这个 skill 面向从 0 到 1 的写作阶段，也面向“大修后重组结构”的情况。它不是泛大纲生成器，而是服务于科研交付的结构设计 skill。

## 触发条件

- "帮我搭这篇论文的结构"
- "这些结果应该怎么组织成论文"
- "给我一个能写的章节大纲"
- "现在这篇稿子结构混乱，帮我重排"

## 核心目标

- 识别论文主问题、主贡献和最强证据
- 确定章节结构和叙事顺序
- 区分哪些部分可以立刻写，哪些必须等结果或补实验
- 给出可执行的写作推进计划

## 输入材料

- 项目简介、研究问题、目标任务
- 结果文件或结果摘要
- 已有稿件、笔记、草稿
- 审稿意见、导师要求、目标期刊或会议约束

## 硬规则

1. 先确认论文的单一主线，不要把多个弱贡献并列成主卖点
2. 章节结构必须围绕证据强弱来排，不要只按“常规模板”机械套
3. 没有结果支撑的部分可以占位，但必须标注为待补证据
4. 如果当前材料不足以确定章节结构，要明确缺什么，不要假装大纲已经成熟
5. 大纲不仅要有章节名，还要写每章的任务和输入证据

## 工作流程

1. 读取项目背景、已有稿件和结果摘要
2. 明确 4 件事：
   - 主研究问题
   - 主贡献
   - 最强证据
   - 当前最大短板
3. 设计论文骨架：
   - 标题方向
   - 摘要主线
   - 引言中的 gap 和贡献
   - 方法章节结构
   - 实验章节结构
4. 为每章标注：
   - 写作目标
   - 依赖证据
   - 当前完成度
   - 下一步动作
5. 输出推进顺序和风险点

## 重点输出

### 1. Storyline

- 论文最该讲的主线是什么
- 不该抢主线的次要点有哪些

### 2. Outline

至少覆盖：

- 标题方向
- 摘要结构
- 引言结构
- 方法结构
- 实验结构
- 讨论 / 结论结构

### 3. Writing Queue

把章节分成三类：

- `ready_to_write`
- `write_with_placeholders`
- `blocked_by_evidence`

## 输出格式

尽量按这个结构输出：

```markdown

## 当前主线
- 研究问题：
- 主贡献：
- 最强证据：
- 当前短板：

## 建议大纲

- 任务：
- 证据依赖：

### 2. 方法
- 任务：
- 证据依赖：

### 3. 实验

- 证据依赖：

- ready_to_write：

```

## 与其他技能的关系

- 如果还没读清论文材料，先走 `paper-reading-doctor`
- 如果某个 claim 是否成立还没确认，交给 `experiment-result-auditor`
- 如果已经有成熟稿件要落地改写，交给 `manuscript-result-sync`
- 如果实验证据还没产出，交给 `experiment-runner-doctor`

## 注意事项

- 不要把“论文应该有什么章节”写成空模板
- 真正重要的是每章服务什么叙事、依赖什么证据、现在能不能写
- 如果目标期刊或会议已有明确约束，要把约束写进结构建议
